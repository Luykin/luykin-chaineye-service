# Supabase 部署详细步骤指南

## 📋 文件说明

### 1. `kong/kong.yml` - Kong API 网关路由配置
**作用**：这是 Kong API 网关的配置文件，用于将所有 Supabase 服务的请求路由到正确的后端服务。

**具体功能**：
- 将 `/auth/v1/*` 路由到 GoTrue 认证服务
- 将 `/rest/v1/*` 路由到 PostgREST API 服务
- 将 `/realtime/v1/*` 路由到 Realtime WebSocket 服务
- 将 `/storage/v1/*` 路由到 Storage 存储服务
- 将 `/functions/v1/*` 路由到 Edge Functions 服务
- 为所有路由启用 CORS（跨域资源共享）

**您需要做什么**：✅ **不需要修改**，这个文件已经配置好了。

---

### 2. `generate-supabase-keys.js` - JWT 密钥生成脚本
**作用**：生成 Supabase 所需的 JWT 密钥和 Token。

**为什么需要**：
- Supabase 使用 JWT（JSON Web Token）进行身份验证
- 需要三个密钥：
  1. `SUPABASE_JWT_SECRET` - 用于签名和验证 JWT
  2. `SUPABASE_ANON_KEY` - 匿名用户的访问令牌
  3. `SUPABASE_SERVICE_KEY` - 服务角色的访问令牌（拥有最高权限）

**您需要做什么**：✅ **必须执行**，这是第一步！

---

### 3. `supabase-init.sql` - 数据库初始化脚本
**作用**：在您的 PostgreSQL 数据库中创建 Supabase 所需的基础结构。

**具体功能**：
- 创建必要的数据库扩展（uuid-ossp, pgcrypto）
- 创建 Supabase 角色（anon, authenticated, service_role）
- 创建必要的 schema（storage, graphql_public, _realtime）
- 设置权限和表结构

**您需要做什么**：✅ **必须执行**，这是第二步！

### 步骤 3：启动 Supabase 服务

1. **确保 Docker 正在运行**：
```bash

```




## 🔧 常见问题排查

### 问题 1：服务启动失败
**解决方案**：
```bash
# 查看详细错误日志
docker-compose -f docker-compose.supabase.yml logs [服务名]

# 重新启动服务
docker-compose -f docker-compose.supabase.yml restart [服务名]
```

### 问题 2：数据库连接失败
**检查**：
- 数据库地址是否正确：`150.5.158.179`
- 数据库端口是否正确：`5432`
- 用户名和密码是否正确
- 防火墙是否允许连接

### 问题 3：端口被占用
**解决方案**：
```bash
# 检查端口占用
lsof -i :8062
lsof -i :3000

# 如果端口被占用，可以修改 .env-pro 中的端口配置
```

### 问题 4：Studio 界面不是中文
**解决方案**：
- 检查 `.env-pro` 中是否添加了中文配置
- 重启 Studio 服务：`docker-compose -f docker-compose.supabase.yml restart studio`
- 清除浏览器缓存后重新访问
